# 💡 为什么有的 HTML 能直接打开，而有的需要服务器？

## 📊 对比说明

| 对比项               | 可以直接打开的 HTML | 需要服务器运行的 HTML（本项目）              |
| -------------------- | ------------------- | -------------------------------------------- |
| **文件内容**         | 纯 HTML 标签        | 引用了外部文件（CSS、JS、JSON 等）           |
| **是否加载外部资源** | ❌ 否               | ✅ 是                                         |
| **是否执行脚本**     | ❌ 无或仅简单内联   | ✅ 有外部 JS 或 `fetch()` 请求                |
| **访问方式**         | `file://`（双击）   | `http://localhost:8000`（HTTP 服务器）       |
| **浏览器安全限制**   | 不受限制            | 受 CORS 策略限制，禁止本地跨文件访问          |
| **常见现象**         | 页面正常显示        | 样式丢失、脚本不执行、JSON 无法加载          |
| **错误提示**         | 无                  | `Access blocked by CORS policy` 或 "资源加载失败" |
| **解决方法**         | 直接打开即可        | 启动本地服务器访问                           |

## 🔐 技术原因

### CORS 策略（跨域资源共享）

浏览器出于安全考虑，禁止通过 `file://` 协议加载外部资源：
- ❌ 无法通过 `fetch()` 加载 JSON 文件
- ❌ 无法跨文件访问外部 CSS/JS
- ✅ 需要通过 HTTP 服务器访问

### 本项目为什么需要服务器？

本项目需要服务器，因为：

1. **加载 JSON 数据** - `script.js` 使用 `fetch('metro_data.json')` 加载数据
2. **外部 CSS/JS** - `index.html` 引用了外部样式和脚本文件
3. **异步请求** - 需要 HTTP 协议支持异步数据加载

## ✅ 解决方案

### 方法一：使用启动脚本（最简单）

**直接双击 `启动网页.bat` 文件**

### 方法二：手动启动服务器

**Python 方式：**
```bash
python -m http.server 8000
```

**Node.js 方式：**
```bash
http-server -p 8000
```

然后在浏览器访问 `http://localhost:8000`

📖 详细启动方法请参考 [`启动方法.md`](启动方法.md)

## 💡 一句话总结

> **纯 HTML 文件可以直接打开**  
> **一旦页面依赖 CSS、JS、JSON 等外部资源，就必须通过 HTTP 服务器访问。**  
> **这是因为浏览器的 CORS 安全策略禁止本地文件之间的跨访问。**
