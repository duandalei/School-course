# 🧱 为什么使用 `index.html` 这个名字？

## 📋 一、为什么不建议改 `index.html` 的名字

| 原因                 | 说明                                                         |
| -------------------- | ------------------------------------------------------------ |
| ⚙️ **默认规则**       | 所有常见服务器（Python、Node.js、Nginx、Apache 等）默认只会在访问目录时自动加载 `index.html`。改名后浏览器访问 `/` 时不会自动打开页面。 |
| 🌐 **兼容性最好**     | 几乎所有网站、托管平台（包括阿里云、GitHub Pages、Vercel、小程序 WebView）都以 `index.html` 作为默认首页。 |
| 🧩 **避免路径混乱**   | 改成其他名字后必须写完整路径访问（如 `/home.html`），链接管理更麻烦。 |
| 🚀 **上线部署更方便** | 小程序、静态网站托管、CDN 等都会默认寻找 `index.html`，保持原名可直接部署，无需额外配置。 |
| 🔍 **搜索引擎友好**   | 搜索引擎会优先索引 `index.html` 作为网站的入口页面。         |

## ✅ 一句话总结

> **"index.html" 是网页世界通用的"首页信号"，**
> **不用配置、兼容性最好、最省心。**

------

## 🛠️ 二、如果你**确实想改名字**（例如改成 `main.html`）

### 1️⃣ 访问方式改变

改名后你必须手动指定完整路径：

```
http://localhost:8000/main.html
```

否则访问 `http://localhost:8000/` 会只显示目录或报错。



## 📊 二、改名后的对比

### ✅ 总结对比表

| 服务器类型                           | 默认首页   | 改名后能否自动加载 | 解决方法（自动加载）                               | 解决方法（手动访问）                             |
| ------------------------------------ | ---------- | ------------------ | -------------------------------------------------- | ------------------------------------------------ |
| **Python (`python -m http.server`)** | index.html | ❌ 否               | 无法自动加载                                       | 在浏览器中访问 `http://localhost:8000/main.html` |
| **Node.js (`http-server`)**          | index.html | ✅ 可配置           | 运行命令时加参数 `--index main.html`               | 在浏览器中访问 `http://localhost:8000/main.html` |
| **生产服务器（Nginx/Apache 等）**    | index.html | ✅ 可配置           | 修改配置文件中默认首页（如 Nginx 的 `index` 指令） | 在浏览器中访问 `http://yourdomain.com/main.html` |

## 💡 三、建议

**强烈建议保持 `index.html` 这个名字，因为：**

- ✅ 所有服务器默认支持
- ✅ 不需要额外配置
- ✅ 兼容性最好
- ✅ 部署最方便
- ✅ 符合行业标准

除非有特殊需求，否则不建议改名。
