# 文件关联说明（成都地铁查询系统）

## 📋 一、文件关系总览

### Web 前端链路

```
启动网页.bat 
  ↓
  启动本地 HTTP 服务器（端口 8000）
  ↓
index.html (语义化标签、ARIA支持)
  ├─→ style.css (CSS变量、响应式设计)
  ├─→ script.js (模块化代码、错误处理)
  └─→ metro_data.json (通过 fetch() 异步加载)
       ↑
       └── 由 convert_data.py 从 成都地铁路径.txt 转换生成
```

### 数据转换链路

```
成都地铁路径.txt (原始数据)
  ↓
convert_data.py (Python脚本，带错误处理)
  ├─→ 读取并解析 TXT 文件
  ├─→ 构建站点索引
  ├─→ 生成邻接表
  └─→ metro_data.json (JSON格式，供Web使用)
```

### C++ 终端版链路

```
成都地铁路径.cpp (优化后的源码)
  ↓
  编译 (g++ -std=c++11)
  ↓
output/成都地铁路径.exe (可执行文件)
  ↓
  运行时读取
  ├─→ 成都地铁路径.txt (当前目录)
  └─→ ../成都地铁路径.txt (上级目录，备用)
```

## 🔄 四、典型工作流程

### Web 版本工作流

#### 正常使用流程

```
1. 运行启动脚本
   ↓
   启动网页.bat
   ↓
2. 浏览器自动打开
   ↓
   http://localhost:8000
   ↓
3. 使用功能
   ├─→ 线路查询
   ├─→ 站点查询
   └─→ 路径规划
```

#### 更新数据流程

```
1. 编辑原始数据
   ↓
   成都地铁路径.txt
   ↓
2. 运行转换脚本
   ↓
   python convert_data.py
   ↓
3. 生成JSON数据
   ↓
   metro_data.json
   ↓
4. 刷新浏览器即可看到更新
```

### C++ 版本工作流

#### 编译运行流程

```
1. 编译源码
   ↓
   g++ 成都地铁路径.cpp -o output/成都地铁路径.exe -std=c++11
   ↓
2. 运行程序
   ↓
   output/成都地铁路径.exe
   ↓
3. 程序查找数据文件
   ├─→ 当前目录/成都地铁路径.txt
   └─→ 上级目录/成都地铁路径.txt
   ↓
4. 命令行交互
   ├─→ 查询线路
   ├─→ 查询站点
   └─→ 查询路径
```

## 💡 五、注意事项

### Web 版本

- ⚠️ **必须通过 HTTP 服务器访问**，不能直接双击打开 HTML 文件
- ✅ 推荐使用 `启动网页.bat` 一键启动
- 📝 更新数据后需要重新运行 `convert_data.py`
- 🔄 数据文件 `metro_data.json` 需要与 HTML 在同一目录

### C++ 版本

- 📁 确保数据文件 `成都地铁路径.txt` 在程序运行目录或上级目录
- 🔧 编译时需指定 C++11 标准：`-std=c++11`
- 💻 支持循环菜单，输入 0 退出程序
- ✅ 完善的输入验证，防止错误输入
